# 📘 Поліноміальна регресія — конспект

## 🎯 Головна ідея (the big picture)

Регресія — це **знаходження формули за даними**.

### Як це працює в реальності:

```
КРОК 1: ТРЕНУВАННЯ (training)
   У нас є старі дані, де ми ЗНАЄМО відповідь:
   площа=50м², район=центр  →  ціна=80 000$
   площа=30м², район=околиця →  ціна=35 000$
   ...100 таких прикладів...

   Модель шукає формулу:
   ціна = w₁·площа² + w₂·район + w₃·площа·район + w₀

   Градієнтний спуск знаходить w₀, w₁, w₂, w₃ → ФОРМУЛА ГОТОВА ✅

КРОК 2: ВИКОРИСТАННЯ (prediction)
   Приходить нова квартира, ціну якої ми НЕ ЗНАЄМО:
   площа=45м², район=центр  →  ціна = ?

   Підставляємо у знайдену формулу → отримуємо ціну! 💰
```

### В нашій лабі:

Ми знаємо "правильну" формулу `y = 3x₁² + 2x₂ + 5x₁x₂ + 1`,  
щоб **перевірити**, чи модель правильно її знайшла.  
В реальності формулу ніхто не знає — є лише дані.

```
Лаба:      дані → модель знаходить weights → порівнюємо з відомою формулою ✓
Реальність: дані → модель знаходить weights → використовуємо для прогнозів 💰
```

---

## 🔁 FOO — загальна структура всіх методів

Кожен метод градієнтного спуску має **однаковий скелет**:

```
1. PolynomialFeatures → створити ознаки (x₁², x₁·x₂, x₂², ...)
2. weights = 0         → стартуємо з нульових коефіцієнтів
3. Цикл (epochs разів):
     y_pred   = X · weights          — передбачення
     error    = y_pred - y           — помилка
     gradient = напрямок виправлення — куди рухатись
     ✨ weights = weights - lr · ...  — ОНОВЛЕННЯ (тут різниця між методами!)
4. return weights
```

**Все, що відрізняє методи — це рядок з ✨**  
Решта — однакова.

---

## 🧩 Методи: від простого до складного

### 1️⃣ GD (Gradient Descent) — базовий

> Рахує градієнт по **всіх** зразках, робить один крок.

```
weights = weights - lr · gradient
```

- Простий і стабільний
- Потребує **багато epochs** (30 000)
- Один `lr` для всіх weights

---

### 2️⃣ SGD (Stochastic Gradient Descent) — випадковий

> Бере **один** випадковий зразок, робить крок по ньому.

```
# Для кожного зразка i:
weights = weights - lr · gradient_i
```

- Швидше "вчиться" на кожному кроці
- Але **шумний** — стрибає туди-сюди
- Вкладений цикл → повільний у Python

| GD | SGD |
|---|---|
| Дивиться на всіх 100 студентів | Дивиться на 1 випадкового |
| Точний, але повільний крок | Швидкий, але неточний крок |

---

### 3️⃣ RMSProp — адаптивний lr

> Проблема GD: один `lr` для всіх weights.  
> RMSProp дає **кожному weight свій lr**.

```
s = β · s + (1-β) · gradient²        ← запам'ятовує "розмір" градієнтів
weights = weights - lr · gradient / √s
```

- Великий градієнт → менший крок (щоб не перестрибнути)
- Маленький градієнт → більший крок (щоб не застрягти)
- Потребує мало epochs (**1 500**) → найшвидший!

---

### 4️⃣ Adam — RMSProp + момент

> Поєднує дві ідеї:
> - **m** (момент) — "інерція", пам'ятає напрямок руху
> - **s** (адаптивний lr) — як у RMSProp

```
m = β₁ · m + (1-β₁) · gradient       ← напрямок (куди рухались)
s = β₂ · s + (1-β₂) · gradient²      ← швидкість (адаптивний lr)

m̂ = m / (1 - β₁ᵗ)                    ← корекція зміщення
ŝ = s / (1 - β₂ᵗ)

weights = weights - lr · m̂ / √ŝ
```

- Найпопулярніший оптимізатор у ML
- Стабільний + адаптивний
- Корекція зміщення — бо на початку m і s ≈ 0

---

### 5️⃣ Nadam — Adam + Nesterov

> Adam дивиться **назад** (куди рухався).  
> Nadam спочатку **стрибає вперед**, потім коригує.

```
m_nesterov = β₁ · m̂ + (1-β₁) · gradient / (1-β₁ᵗ)

weights = weights - lr · m_nesterov / √ŝ
```

- "Погляд у майбутнє" — трохи точніший за Adam
- На практиці різниця невелика

---

## ⚡ Порівняння

| Метод | Epochs | Час (сек) | Ідея |
|-------|--------|-----------|------|
| **GD** | 30 000 | 0.058 | Простий, по всіх даних |
| **SGD** | 3 000 | 0.868 | По одному зразку, шумний |
| **RMSProp** | 1 500 | **0.007** | Адаптивний lr → найшвидший |
| **Adam** | 5 000 | 0.030 | Момент + адаптивний lr |
| **Nadam** | 5 000 | 0.037 | Adam + погляд вперед |

---

## 🧠 Як запам'ятати?

```
GD      →  просто йди вниз
SGD     →  йди вниз, але навмання
RMSProp →  йди вниз, підлаштовуй кроки
Adam    →  йди вниз з інерцією + підлаштовуй кроки
Nadam   →  Adam, але дивись вперед
```

Еволюція:  
**GD** → додаємо випадковість → **SGD**  
**GD** → додаємо адаптивний lr → **RMSProp**  
**SGD** (момент) + **RMSProp** → **Adam**  
**Adam** + Nesterov → **Nadam**

---

## 📌 Ключові терміни

| Термін | Що це |
|--------|-------|
| `lr` (learning rate) | Розмір кроку |
| `epochs` | Скільки разів пройти по даних |
| `gradient` | Напрямок найбільшого зростання помилки |
| `weights` | Коефіцієнти, які шукаємо |
| `PolynomialFeatures` | Створює x², x₁x₂, ... з x₁, x₂ |
| `β` (beta) | Скільки "пам'яті" зберігати (0.9 = 90% старого) |
| `ε` (epsilon) | Маленьке число, щоб не ділити на 0 |

---

## 💡 Головний висновок

Всі методи роблять **одне й те саме** — шукають weights, що мінімізують помилку.  
Різниця лише в тому, **як саме** вони оновлюють weights на кожному кроці.  
Адаптивні методи (RMSProp, Adam) потребують менше epochs →  
→ швидше знаходять відповідь, навіть якщо кожен крок складніший.
